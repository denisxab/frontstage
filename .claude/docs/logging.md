# Логирование FrontStage

Реализовано в `src/logger.py`. Настраивается через `.env`:

| Переменная  | По умолчанию               | Описание                                     |
| ----------- | -------------------------- | -------------------------------------------- |
| `LOG_FILE`  | `/app/logs/frontstage.log` | Путь к файлу внутри контейнера               |
| `LOG_LEVEL` | `INFO`                     | Уровень: `DEBUG`, `INFO`, `WARNING`, `ERROR` |

Файл пробрасывается через Docker volume `./logs:/app/logs`. Ротация: 10 МБ, 5 файлов.

## Категории логгеров

| Имя логгера          | Что логирует                                                 |
| -------------------- | ------------------------------------------------------------ |
| `frontstage.startup` | Старт/стоп приложения, загрузка каталога                     |
| `frontstage.api`     | JSON API запросы (`/api/*`) — INFO для write, DEBUG для read |
| `frontstage.ui`      | HTML-страницы — WARNING при 404, INFO при логине             |
| `frontstage.plugins` | Планировщик: fetch, ошибки, обновление кеша                  |

Просмотр в реальном времени: `invoke logs` или `tail -f logs/frontstage.log`.
